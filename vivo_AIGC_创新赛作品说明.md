# vivo AIGC 创新赛 · 应用赛道 作品说明

---

## 作品名称

**智研笔记（ResearchMind）—— 基于大模型的智能学术研究全流程协作平台**

---

## 作品概述

智研笔记（ResearchMind）面向研究生、科研人员及学术工作者，解决学术研究中文献管理分散、笔记整理低效、知识难以系统化沉淀等核心痛点。平台以 DeepSeek 大模型为智能底座，提供论文上传解析、AI 智能摘要生成、知识图谱可视化、Markdown 笔记协作、论文查重与相似度分析、研究趋势预测、语音交互及摄像头图像分析等一站式功能。核心创新在于将大模型能力深度融入学术研究的每一个环节——从文献入库的自动解析，到笔记写作的 AI 辅助，再到跨论文的知识关联与趋势洞察，形成"上传→解析→关联→写作→分析"的完整闭环。平台采用 React + TypeScript 前端与 Node.js + SQLite 后端架构，支持多用户数据隔离，部署简单开箱即用。

---

## 核心创新点

将大模型 AI 能力从"对话助手"升级为学术研究的全流程智能引擎——论文上传即自动解析生成结构化摘要，知识图谱自动发现跨论文的主题关联，研究趋势分析通过 K-Means 聚类揭示领域演化路径，配合语音输入与摄像头拍照分析，让研究者在一个平台内完成从文献管理到知识产出的完整闭环。

---

## 设计背景与洞察

在研究生群体中，一个反复出现的现象是：研究者每天花费大量时间在"文献管理"和"笔记整理"这两个环节，却很少从中获得真正的知识积累。我们在用户调研中发现三个核心问题：

1. **信息碎片化**：论文分散在不同文件夹、不同平台，笔记格式不统一，知识无法沉淀为结构化的个人知识库。
2. **分析效率低**：面对几十甚至上百篇论文，人工梳理研究脉络、发现热点趋势极其耗时；判断两篇论文的相似度或是否存在抄袭更需要逐字对比。
3. **工具割裂**：论文管理用一个工具、笔记用另一个、AI 对话又是一个、查重还得换平台，工具之间的切换严重打断研究思路。

这些痛点指向一个核心洞察：**研究者需要的不是又一个 AI 对话框，而是一个让 AI 能力贯穿研究全流程的智能工作台。**

## 理念贯穿性

"AI 贯穿全流程"的理念体现在产品的每一层设计中：

- **功能流程**：从首页引导用户上传论文开始，AI 自动解析并生成摘要（/analyze）→ 系统构建知识图谱展示论文间的关键词关联（/graph）→ 用户在研究工作台用 Markdown 整理笔记并获得 AI 辅助（/workbench）→ 通过查重功能比对论文相似度（/similarity）→ 利用趋势分析洞察研究方向演变（/trends）→ 随时在智能助手浮窗中进行 AI 问答和语音/拍照交互。整个流程形成闭环。
- **界面设计**：全局采用统一的蓝色渐变（from-blue-500 to-cyan-500）视觉语言，粒子动画背景贯穿所有页面，营造"科技+学术"的沉浸氛围。
- **交互设计**：AI 助手以右下角浮窗形式常驻，配合会动的眼睛动画和鼠标跟随，让 AI 的存在感不再生硬；语音输入和摄像头拍照降低了文字交互的门槛。

## 作品的界面设计

### （1）整体视觉风格与色彩体系

- **视觉风格**：现代科技学术风——以极简布局为基础，融入粒子动画背景（ParticleAnimation 组件），营造未来感与沉浸感。
- **色彩体系**：
  - 主色调：蓝色渐变（#3B82F6 → #06B6D4），传达专业、信任、科技感
  - 背景层：浅蓝灰色调（slate-50 ~ blue-50），保证文字可读性的同时营造空间层次
  - 强调色：绿色（成功状态）、红色（高相似度警告）、橙色（中等相似度提示）
  - 文字色：深灰色系（slate-700 ~ slate-900），确保长文本阅读舒适
- **设计理由**：学术研究场景需要长时间集中注意力，低饱和度蓝灰色系减少视觉疲劳；粒子动画增加科技感但不喧宾夺主；统一的渐变色语言让 8 个功能页面保持视觉一致性。

### （2）关键界面展示

**首页（/）**：大标题 ResearchMind 蓝色渐变文字 + 粒子动画背景 + "科研新体验，让研究更高效"标语。下方展示 6 大核心功能卡片（论文管理、AI 分析、知识图谱、笔记协作、论文查重、趋势分析），点击即可进入对应功能。

**智慧大屏（/dashboard）**：数据看板式布局，顶部 4 张统计卡片（论文总数、笔记总数、月增长率、关键词数），下方展示最近论文列表、按年份分布的论文数量图表，以及关键词词云，帮助用户一目了然地掌握自己的研究资产。

**知识图谱（/graph）**：D3.js 力导向图将用户的论文库可视化为节点-边网络，节点大小反映关键词权重，边的粗细表示论文间的关联强度。12 列网格布局，左侧浮动导航栏支持快速定位论文段落，关键词高亮与 IntersectionObserver 自动追踪当前阅读位置。

**论文查重（/similarity）**：三种模式——两篇论文对比、全库查重、关键词提取。对比结果用环形进度条展示相似度百分比，高/中/低/不相似四级颜色标签清晰标注风险等级。

**研究趋势（/trends）**：多视图分析——时间线折线图展示关键词演变、K-Means 散点图展示论文聚类、网络图展示论文关联，三个维度帮助研究者洞察领域趋势。

**智能助手浮窗**：右下角悬浮，可展开/收起。顶部展示两只会跟随鼠标移动的 SVG 动画眼睛，增加趣味性和亲和力。底部输入框左侧摄像头按钮、右侧语音按钮，支持多模态交互。

## 作品的交互设计

### （1）核心用户操作流程

**流程一：从论文上传到知识图谱构建（完整研究闭环）**

```
Step 1: 进入"论文分析"页面（/analyze）
        → 拖拽或选择 PDF/TXT/DOCX 文件上传
        → 系统自动解析文件提取文本
        → AI 生成结构化摘要和关键词
        → 论文入库，自动关联到当前用户

Step 2: 进入"知识图谱"页面（/graph）
        → 系统基于所有论文的关键词自动构建力导向图
        → 节点代表关键词，连线代表论文间的关联
        → 点击节点查看关联论文详情
        → 左侧浮动导航栏可快速定位到具体论文段落

Step 3: 进入"研究趋势"页面（/trends）
        → 系统对论文库进行 K-Means 聚类
        → 时间线图展示关键词在不同年份的热度变化
        → 散点图展示论文聚类分布
        → 网络图展示论文间的相似度关系
```

**流程二：与 AI 智能助手的多模态交互**

```
Step 1: 点击右下角 AI 助手图标，展开浮窗
        → 眼睛动画跟随鼠标移动，增加亲和感

Step 2a（文字模式）: 在输入框输入问题，如"帮我总结这篇论文的核心观点"
        → 发送后助手实时流式返回 AI 回答（SSE）
        → 回答基于用户论文库上下文，个性化推荐

Step 2b（语音模式）: 点击麦克风按钮
        → 浏览器开始录音，输入框边框变红色（脉冲动画提示）
        → 实时识别语音转文字显示在输入框
        → 再次点击或发送自动停止录音

Step 2c（拍照模式）: 点击摄像头按钮
        → 打开摄像头画面（优先后置摄像头）
        → 点击"拍照"按钮捕捉画面
        → 预览图片并可选择"重新拍照"或"发送分析"
        → AI 大模型自动识别图片内容（图表数据、公式、文字信息等）
        → 图片与 AI 分析结果同时展示在对话气泡中
```

### （2）交互设计创新之处

- **眼睛动画（Eyes 组件）**：SVG 绘制的眼睛会根据鼠标位置实时计算瞳孔偏移角度和距离，实现自然的"注视跟随"效果。这不只是装饰——它让 AI 助手从一个冰冷的对话框变成了一个有"生命感"的存在，显著提升了用户的交互意愿。
- **语音交互（Web Speech API）**：直接在浏览器端实现中文语音识别（zh-CN），无需安装插件。录音状态用红色脉冲边框视觉反馈，中间结果实时显示，支持连续识别，让双手不便打字的场景（如翻阅纸质文献时）也能顺畅使用 AI 助手。
- **摄像头拍照 AI 识别**：使用 getUserMedia API 调用设备摄像头，Canvas 画布自动压缩到 800×600 分辨率后传输给 AI 大模型。大模型基于视觉能力自动识别图片中的图表数据、公式、文字信息等内容，并结合用户提问给出结构化分析。适用于拍摄纸质文献中的实验数据图表、手写公式推导、会议白板内容等场景。
- **拖拽导航栏（Header）**：顶部导航栏支持鼠标拖拽滚动，并在左右两端显示箭头按钮，在小屏设备上也能流畅切换 8 个功能页面。
- **知识图谱浮动导航**：图谱页面的导航栏使用 IntersectionObserver 自动高亮当前可视区域对应的论文，实现"滚动即定位"的无缝体验。

## 大模型的具体应用说明

智研笔记通过以下方式深度应用 DeepSeek 大模型：

### 1. 论文智能解析（/api/analyze/upload）
用户上传论文文件后，系统提取全文文本，调用 DeepSeek API 生成结构化摘要、关键词列表和研究方法识别。AI 生成的内容自动写入数据库，成为知识图谱和趋势分析的数据基础。

### 2. AI 学术助手对话（/api/assistant/chat）
基于 SSE（Server-Sent Events）流式传输实现打字机效果的实时对话。核心设计：
- **上下文注入**：自动将用户论文库的标题、摘要、关键词注入 system prompt，使 AI 回答能够引用用户已有的研究成果
- **多轮对话记忆**：前端维护对话历史，每次请求携带完整 context，支持连续追问
- **IP 限流**：每 IP 每分钟 30 次请求上限，防止 API 滥用

### 3. 论文查重与相似度分析（/api/similarity）
自研文本分析引擎，结合 AI 与传统 NLP 技术：
- **中文分词**：滑动窗口 Bigram 分词算法，适配学术中文文本
- **TF-IDF 向量化**：将论文文本转换为 TF-IDF 特征向量
- **余弦相似度**：计算向量夹角，量化论文间相似程度
- **段落级查重**：将论文拆分为段落，逐段比对，精确定位高相似内容

### 4. 研究趋势分析（/api/trends）
综合应用 AI 与机器学习技术：
- **K-Means 聚类**：自动将论文库按内容相似度分组，发现隐含的研究方向
- **肘部法则**：自动选择最优聚类数 K
- **关键词时间线**：追踪各研究主题在不同年份的热度变化
- **网络关系图**：基于相似度构建论文间的关联网络

### 5. 笔记 AI 辅助写作
研究工作台的 Markdown 编辑器配合 AI 助手，用户可以边写笔记边向 AI 咨询，AI 基于当前论文上下文提供写作建议、内容扩展和学术表达润色。

### 6. 多模态图片理解（AI Assistant 摄像头模式）
用户通过 AI 助手的摄像头功能拍摄图片后，图片以 base64 编码通过 OpenAI Vision 格式（`image_url`）发送给支持视觉理解的大模型。大模型能够：
- **图表识别**：自动识别并解读折线图、柱状图、散点图中的数据趋势和关键数值
- **公式提取**：识别手写或印刷体数学公式、化学方程式，并以 LaTeX 格式输出
- **文字 OCR**：提取图片中的文字内容，支持中英文混合识别
- **学术分析**：结合图片内容与用户提问，给出专业的学术解读和研究建议

该功能通过 `getUserMedia` API 获取摄像头画面，Canvas 自动压缩至 800×600 后以 SSE 流式方式返回分析结果，实现"拍摄即理解"的实时体验。

## 创新点详细阐释

### 1. 解决方案创新：学术研究全流程 AI 闭环

市面上的学术工具大多聚焦单一环节——Zotero 做文献管理、CNKI 做查重、ChatGPT 做问答。智研笔记的创新在于将 AI 能力注入学术研究的完整生命周期：

```
论文上传 → AI 自动解析 → 知识图谱关联 → 笔记写作辅助 → 论文查重比对 → 趋势洞察分析
```

每一步的产出自动成为下一步的输入，形成数据飞轮——论文越多，图谱越丰富，查重越准确，趋势分析越有洞察力。

### 2. 交互创新：多模态 AI 助手

突破传统文本对话框的局限，实现三种交互模式：
- **文字对话**：基于论文库上下文的个性化 AI 问答
- **语音输入**：浏览器原生 SpeechRecognition，中文语音实时转文字
- **摄像头拍照**：调用设备摄像头拍摄纸质文献或实验数据，AI 大模型自动识别图片中的图表、公式、文字等内容并给出结构化分析

眼睛动画跟随鼠标更是将"冷冰冰的 AI"变得有温度，提升用户情感连接。

### 3. 功能性能创新：自研文本分析引擎

论文查重不依赖第三方查重 API，而是自研完整的分析引擎：
- **滑动窗口 Bigram 分词**：比传统词典分词更适应学术新词和专业术语
- **TF-IDF + 余弦相似度**：经典但高效的向量空间模型
- **段落级精细查重**：不仅给出整体相似度，还精确到具体段落，标注匹配来源
- **K-Means 聚类 + 肘部法则**：自动发现论文库中的隐含研究方向

### 4. 知识图谱创新：12 列网格 + 浮动导航

知识图谱页面采用创新的 12 列响应式网格布局：
- 力导向图占 9 列，论文详情占 3 列
- 浮动导航栏使用 IntersectionObserver 自动高亮当前可视区域
- 关键词高亮定位，一键跳转到论文中的匹配段落
- 拖拽导航栏 + 箭头按钮，适配不同屏幕尺寸

### 5. 架构创新：轻量级全栈方案

- **SQLite 文件数据库**：零配置部署，单文件备份，单用户场景下性能优异
- **Express + Vite 代理**：开发环境一键启动，生产环境前端静态文件由 Express 托管
- **Zustand 持久化状态**：比 Redux 更轻量，自动同步到 localStorage，刷新不丢失
- **JWT 无状态认证**：无需 Redis 存储会话，适合单服务器部署场景

| 痛点 | 严重程度 | 频率 | 智研笔记如何解决 |
|------|---------|------|----------------|

### （3）典型应用场景

**场景一：研究生开题准备**
小王是一名计算机科学专业研一学生，即将开题。他有 50 篇下载的论文 PDF 需要阅读和整理。使用智研笔记：
1. 批量上传论文，AI 自动生成每篇的摘要和关键词
2. 进入知识图谱页面，直观看到"深度学习""自然语言处理""计算机视觉"等主题的关联网络
3. 使用趋势分析发现"大语言模型"相关论文在近两年呈爆发式增长
4. 在研究工作台中用 Markdown 整理文献综述笔记，遇到疑问随时问 AI 助手
最终，小王在 2 天内完成了开题报告的文献综述部分。

**场景二：论文写作与查重**
小李正在撰写毕业论文，需要确保自己的论文没有与已有文献过度相似。使用智研笔记：
1. 上传自己的草稿到查重页面，与论文库中的 30 篇参考文献逐一对比
2. 系统标红了 3 个高相似度段落，精确定位到具体句子
3. 小李修改后再次查重，相似度从 35% 降到 12%
4. 写作过程中用语音功能向 AI 助手咨询某个方法论的描述方式

**场景三：拍照识别实验数据**
小张在实验室看到一篇论文中的关键图表，想快速了解其中的数据趋势：
1. 打开 AI 助手浮窗，点击摄像头按钮
2. 对准论文中的折线图拍照，点击"发送分析"
3. AI 大模型自动识别图片中的坐标轴、数据点和趋势线，输出结构化分析
4. 小张追问"这个趋势说明了什么研究意义"，AI 基于论文上下文给出专业解读

### （4）社会价值

- **提升科研效率**：将研究者从繁琐的文献管理和笔记整理中解放出来，把更多时间投入到真正的思考和创新中。
- **降低学术门槛**：AI 辅助解析和总结让研究生新生也能快速理解复杂的学术论文，缩短从入门到产出的成长周期。
- **促进知识积累**：结构化的知识图谱让个人知识不再碎片化，形成可复用、可检索的个人学术资产库。
- **维护学术诚信**：内置查重功能帮助研究者在写作阶段就发现潜在的重复问题，从源头上减少无意识抄袭。
- **推动 AI 赋能教育**：将大模型能力从通用对话场景深入到垂直的学术研究领域，探索 AI + Education 的落地路径。

## 市场欢迎程度

### 与行业解决方案的对比优势

| 对比维度 | 智研笔记 ResearchMind | Zotero/EndNote | CNKI 查重 | ChatGPT/DeepSeek 对话 |
|---------|---------------------|---------------|----------|---------------------|
| **功能完整性** | 论文管理 + AI 分析 + 笔记 + 查重 + 趋势 + 语音 + 拍照 | 仅论文管理 | 仅查重 | 仅 AI 对话 |
| **AI 深度整合** | AI 贯穿全流程，论文上下文自动注入 | 无 AI 能力 | 规则匹配，无 AI | 无论文上下文 |
| **知识图谱** | 自动构建，力导向图可视化 | 无 | 无 | 无 |
| **趋势分析** | K-Means 聚类 + 时间线 + 网络图 | 无 | 无 | 无 |
| **多模态交互** | 文字 + 语音 + 摄像头拍照 | 无 | 无 | 仅文字 |
| **数据隐私** | 本地 SQLite，数据不离开设备 | 云端同步 | 上传到服务器 | 数据发送到云端 |
| **部署成本** | 零配置，SQLite 文件数据库 | 需要安装客户端 | 按次付费 | 需要 API Key |
| **用户体验** | 统一平台，闭环流程 | 需要配合其他工具 | 独立使用 | 无法管理论文 |

### 核心竞争优势

1. **全流程闭环**：市场上没有一个产品能同时覆盖文献管理、AI 分析、知识图谱、笔记协作、论文查重和趋势分析。智研笔记将这些能力整合到一个平台中。
2. **AI 深度嵌入**：不同于"外挂 ChatGPT"的简单集成，AI 能力直接嵌入每个功能模块——上传即解析、写笔记即辅助、查重即分析。
3. **多模态创新**：语音输入和摄像头拍照 AI 图片识别交互在学术工具中极为罕见。大模型的视觉理解能力让研究者可以拍摄纸质文献中的图表、公式、实验数据，AI 实时识别并给出结构化分析，为翻阅纸质资料、实验室数据采集等场景提供了全新的交互方式。
4. **技术自研**：查重引擎（TF-IDF + 滑动窗口分词 + 余弦相似度）和聚类分析（K-Means + 肘部法则）均为自研实现，不依赖第三方 API，降低使用成本和数据泄露风险。
5. **适配 vivo 生态**：轻量级架构（React + Node.js + SQLite）具备良好的移动端适配潜力，可作为 vivo 平板或手机端的学术助手应用落地。

### 发展趋势契合

- AI + Education 是当前最受关注的垂直领域之一
- 大模型从通用对话向垂直行业深度应用是明确趋势
- 科研工具的智能化和一体化是用户的核心期待
- 轻量化、隐私优先的本地化部署方案越来越受青睐
